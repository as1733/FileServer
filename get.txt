<p>Confused by Node's filesystem parsing. Here's my code:</p>

<pre><code>var fs = require('fs'),
    xml2js = require('xml2js');

var parser = new xml2js.Parser();

var stream = fs.createReadStream('xml/bigXML.xml');
stream.setEncoding('utf8');

stream.on('data', function(chunk){ 

    parser.parseString(chunk, function (err, result) {
        console.dir(result);
        console.log('Done');
    });
});


stream.on('end', function(chunk){
    // file have been read over,do something...
    console.log("IT'S OVER")
});
</code></pre>

<p>This causes...nothing to happen. No output from XML2JS/the parser at all. When I try to <code>console.log(chunk)</code> it seems that the <code>chunks</code> aren't being output in any sort of meaningful chunks based on anything other than perhaps byte size. The output for one 'chunk' is:</p>

<pre><code>&lt;?xml version="1.0" encoding="UTF-8"?&gt;
    &lt;merchandiser xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="merchandiser.xsd"&gt;
    &lt;header&gt;&lt;merchantId&gt;1237&lt;/merchantId&gt;&lt;merchantName&gt;NORDSTROM.com&lt;/merchantName&gt;&lt;createdOn&gt;12/13/2013 23:50:57&lt;/createdOn&gt;&lt;/header&gt;
    &lt;product product_id="52863929"&gt;// product info&lt;/product&gt;
    &lt;product product_id="26537849"&gt;// product info&lt;/product&gt;
    &lt;product product_id="25535647"&gt;// product info&lt;/product&gt;
</code></pre>

<p>This chunk has lots and lots of <code>&lt;product&gt;</code> entries from the XML inside of it. The chunk will end somewhere in the middle of a <code>&lt;product&gt;</code> entry and the next chunk will begin from where this left off.</p>

<p>The main question is <strong>How do I get the <code>createReadStream</code> to output chunks starting at <code>&lt;product</code> and ending at <code>&lt;/product&gt;</code>?</strong></p>

<p>EDIT: for the purposes of getting the proper output, here's what the XML from the beginning to the end of the first <code>&lt;product&gt;</code> looks like:</p>

<pre><code>&lt;?xml version="1.0" encoding="UTF-8" ?&gt;
&lt;merchandiser xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="merchandiser.xsd"&gt;

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You have two possibilities to tackle your issue.</p>

<p>As stated by damphat, XML2JS needs the full XML content before it can parse the data. But you have a file stream, which, well, streams data chunk by chunks. The first solution is to convert this stream of data into a nice big Buffer, and then send it to XML2JS. For this purpose, you can use the <a href="https://npmjs.org/package/stream-to" rel="noreferrer"><code>stream-to</code> package</a> (<code>npm i stream-to</code>) which will convert the file stream into an array of buffers, which we'll then concatenate into one single buffer using <a href="http://nodejs.org/docs/latest/api/buffer.html#buffer_class_method_buffer_concat_list_totallength" rel="noreferrer"><code>Buffer.concat</code></a>, like this:</p>

<pre><code>var fs = require('fs')
var streamTo = require('stream-to')
var xml2js = require('xml2js')

var file = fs.createReadStream('input.xml')

streamTo.array(file, function (err, arr) {
    if (err) return console.log(err.message)

    var content = Buffer.concat(arr)
    var parser = new xml2js.Parser()
    parser.parseString(content, function (err, res) {
        if (err) return console.log(err.message)
        console.log(res.merchandiser.product)
    })
})
</code></pre>

<p>This works quite well, but since it needs to hold the full file into memory, it won't work if your input files are really big. To handle really big files, you need to use a streaming XML parser, such as <code>sax</code>. However <code>sax</code> doesn't create Javascript objects, but is an EventEmitter, and is a bit harder to use since you have to handle all relevant events to build your object on the fly.</p>

<p>You can use for instance the <a href="https://npmjs.org/package/saxpath" rel="noreferrer">SaXPath library</a>, which supports a small subset of the XPath syntax. This library emits a <code>match</code> event every time it matches the XPath pattern. Here's an example:</p>

<pre><code>var saxpath = require('saxpath')
var fs = require('fs')
var sax = require('sax')

var saxParser = sax.createStream(true)
var streamer = new saxpath.SaXPath(saxParser, '/merchandiser/product')

streamer.on('match', function(xml) {
    console.log(xml);
});

fs.createReadStream('input.xml').pipe(saxParser)
</code></pre>

<p>You then have two options:</p>

<ol>
<li>Since you now have the XML that matches only one product at a time, you can use <code>xml2js</code> to parse a single product at a time</li>
<li>SaXPath supports multiple recorders: the default recorder listens to sax events and re-creates the corresponding XML (which is what allowed us to use the first solution), but you can roll out your own recorder, that listens to sax events and creates on the fly javascript objects.</li>
</ol>

<p>xml2js is for full loaded xml.</p>

<p>In your case using sax, it is a stream parser:</p>

<p>// install</p>

<pre><code>npm install sax
</code></pre>

<p>// this code is for print all product_id</p>

<pre><code>var fs = require('fs');
var sax = require('sax');

var saxStream = sax.createStream();

saxStream.onopentag = function (node) {
    if(node.name === 'PRODUCT'){
        console.log(node.attributes.PRODUCT_ID);
    }
};

fs.createReadStream('xml/bigXML.xml').pipe(saxStream);
</code></pre>

<p>ouput:</p>

<pre><code>52863929
26537849
25535647
</code></pre>
